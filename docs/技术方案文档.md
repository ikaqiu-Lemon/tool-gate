## 1 技术背景

### 1.1 Claude Code 插件系统

Claude Code 插件是一个自包含的目录，可打包 skills、hooks、agents、MCP servers、LSP servers 等组件。插件通过 `.claude-plugin/plugin.json` 清单声明元数据和组件路径。

**关键规范要点**（参考 [Plugins Reference](https://code.claude.com/docs/en/plugins-reference)、[ivan-magda/claude-code-plugin-template](https://github.com/ivan-magda/claude-code-plugin-template)）：

- `plugin.json` 是唯一放在 `.claude-plugin/` 目录中的文件
- skills/、hooks/、agents/、.mcp.json 放在插件根目录
- 插件安装后，Claude Code 会话启动时自动连接插件声明的 MCP Server
- 插件的 hooks 与用户的 hooks 并行执行
- 插件中的 skills 自动以 `plugin-name:skill-name` 命名空间注册
- `${CLAUDE_PLUGIN_ROOT}` 指向插件根目录，`${CLAUDE_PLUGIN_DATA}` 指向持久数据目录

### 1.2 Claude Code Hooks 系统

Hooks 是 Claude Code 在关键生命周期点触发的事件处理器（参考 [Hooks Reference](https://code.claude.com/docs/en/hooks)）。本项目使用的核心事件：

| 事件 | 触发时机 | 可用能力 |
|------|----------|----------|
| **SessionStart** | 会话开始 | 注入 additionalContext |
| **UserPromptSubmit** | 用户消息发送后、模型响应前 | 注入 additionalContext（**对应 Skill-Hub 的 wrap_model_call，每轮触发**） |
| **PreToolUse** | 工具调用前 | allow/deny/ask + updatedInput + additionalContext |
| **PostToolUse** | 工具调用成功后 | additionalContext / block |

Hook 输入通过 stdin（command 类型）或 HTTP POST body 接收 JSON，输出通过 stdout 返回 JSON。

**PreToolUse 输出格式**：

```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "allow|deny|ask",
    "permissionDecisionReason": "说明文字",
    "updatedInput": {},
    "additionalContext": "额外上下文注入给模型"
  }
}
```

### 1.3 MCP (Model Context Protocol)

MCP Server 通过标准协议向 Claude 暴露工具。本项目使用 stdio 模式的 Python MCP Server（基于 [MCP Python SDK](https://github.com/modelcontextprotocol/python-sdk)），要求 Python 3.10+、SDK 1.2.0+。

**关键约束**：Claude Code 对 MCP `tools/list_changed` 动态刷新的支持尚不完全稳定。因此本项目采用**固定元工具面 + 服务端子能力分发**策略。

### 1.4 Skill-Hub 项目经验

本项目的治理机制直接源自 Skill-Hub 项目的实践（详见 `Skill-Hub项目准备版.md`、`手写 Skills 长文版.md`）：

- **双平面架构**：知识平面（read_skill）与执行平面（enable_skill）分离
- **Middleware 治理点**：在模型调用前拦截并重写 tools 和 prompt
- **状态驱动的工具重写**：active_tools 每轮基于当前 skills_loaded 重算
- **LRU/TTL 缓存**：技能元数据和文档内容分层缓存

## 2 总体架构

### 2.1 三层架构

```
┌──────────────────────────────────────────────────────┐
│                  Claude Code 宿主                      │
│  ┌────────┐  ┌────────┐  ┌──────────┐  ┌──────────┐ │
│  │ 模型   │  │ 内置   │  │ 其他MCP  │  │ 其他插件 │ │
│  │        │  │ 工具   │  │ Server   │  │          │ │
│  └───┬────┘  └────────┘  └──────────┘  └──────────┘ │
│      │                                                │
│  ┌───┴────────────────────────────────────────────┐  │
│  │             插件适配层 (Plugin Layer)            │  │
│  │  ┌──────────┐  ┌───────────┐  ┌─────────────┐ │  │
│  │  │ MCP      │  │ Hooks     │  │ Skills      │ │  │
│  │  │ Server   │  │ Handler   │  │ (SKILL.md)  │ │  │
│  │  │ (6 元工具)│  │ (事件拦截)│  │ (知识入口)  │ │  │
│  │  └────┬─────┘  └─────┬─────┘  └─────────────┘ │  │
│  └───────┼──────────────┼─────────────────────────┘  │
│          │              │                             │
│  ┌───────┴──────────────┴─────────────────────────┐  │
│  │           治理核心层 (Governance Core)           │  │
│  │  ┌────────────┐  ┌──────────┐  ┌────────────┐ │  │
│  │  │ Skill      │  │ State    │  │ Policy     │ │  │
│  │  │ Indexer    │  │ Manager  │  │ Engine     │ │  │
│  │  ├────────────┤  ├──────────┤  ├────────────┤ │  │
│  │  │ Tool       │  │ Grant    │  │ SQLite     │ │  │
│  │  │ Rewriter   │  │ Manager  │  │ Store      │ │  │
│  │  └────────────┘  └──────────┘  └────────────┘ │  │
│  └────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────┘
```

**层次职责**：

1. **插件适配层**：负责与 Claude Code 宿主通信。MCP Server 暴露元工具供模型调用；Hooks Handler 处理生命周期事件；Skills 提供知识平面入口。
2. **治理核心层**：纯 Python 逻辑，不依赖 Claude Code 特定 API。负责技能索引、状态管理、策略评估、工具重写、授权管理、持久化。与宿主平台解耦，未来可复用于其他编辑器。

### 2.2 数据流

一次完整的治理流程数据流如下：

```
[SessionStart]
    → hook_handler.py 接收事件 JSON (stdin)
    → state_manager.load_or_init(session_id)
    → grant_manager.cleanup_expired()
    → skill_indexer.build_index()
    → tool_rewriter.generate_context_summary()
    → 返回 {additionalContext: "技能目录摘要"} (stdout)

[模型调用 list_skills]
    → mcp_server.py 接收 MCP 请求
    → skill_indexer.list_skills()
    → 返回技能元数据列表

[模型调用 read_skill("db-ops")]
    → skill_indexer.read_skill("db-ops")
    → 检查 LRU/TTL 缓存 → 命中则直接返回
    → 未命中则读取 SKILL.md，解析后缓存
    → 返回完整 SOP

[模型调用 enable_skill("db-ops", scope="session", ttl=3600)]
    → policy_engine.evaluate("db-ops", session_state)
    → 风险等级 → low → auto grant
    → grant_manager.create_grant(...)
    → state_manager.add_to_skills_loaded("db-ops")
    → tool_rewriter.recompute_active_tools(session_state)
    → 返回 {granted: true, allowed_tools: [...]}

[模型调用 run_skill_action("db-ops", "query", {sql: "..."})]
    → 验证 grant 有效性
    → 验证 op 在 allowed_ops 中
    → 分发到执行逻辑
    → 返回执行结果

[PreToolUse Hook]
    → hook_handler.py 接收事件 JSON
    → state_manager.get_active_tools(session_id)
    → tool_name in active_tools?
        → 是 → {permissionDecision: "allow"}
        → 否 → {permissionDecision: "deny", reason: "..."}
```

## 3 模块设计

### 3.1 数据模型（`src/tool_governance/models/`）

所有数据模型使用 Pydantic v2 BaseModel 定义。

#### 3.1.1 Skill 模型（`skill.py`）

```python
from pydantic import BaseModel

class StageDefinition(BaseModel):
    """技能内某个阶段的定义"""
    stage_id: str                          # 阶段标识（如 "analysis", "execution"）
    description: str                       # 阶段说明
    allowed_tools: list[str]               # 该阶段允许的工具

class SkillMetadata(BaseModel):
    """技能元数据，从 SKILL.md frontmatter 解析"""
    skill_id: str                          # 唯一标识
    name: str                              # 显示名称
    description: str                       # 简短描述
    risk_level: Literal["low", "medium", "high"] = "low"
    allowed_tools: list[str] = []          # 白名单工具列表（无 stage 时使用）
    allowed_ops: list[str] = []            # 允许的操作名
    stages: list[StageDefinition] = []     # 阶段定义（可选，有 stage 时按 stage 取 allowed_tools）
    default_ttl: int = 3600                # 默认 TTL (秒)
    source_path: str                       # SKILL.md 文件路径
    version: str = "1.0.0"                 # 版本号

class SkillContent(BaseModel):
    """技能完整内容（read_skill 返回）"""
    metadata: SkillMetadata
    sop: str                               # SOP 正文 (Markdown)
    examples: list[str] = []               # 使用示例
```

#### 3.1.2 Grant 模型（`grant.py`）

```python
from datetime import datetime
from pydantic import BaseModel

class Grant(BaseModel):
    """一次 enable_skill 创建的授权记录"""
    grant_id: str                          # UUID
    session_id: str
    skill_id: str
    allowed_ops: list[str]                 # 该 grant 授权的操作
    scope: Literal["turn", "session"] = "session"
    ttl_seconds: int
    status: Literal["active", "expired", "revoked"] = "active"
    granted_by: Literal["auto", "user", "policy"] = "auto"
    reason: str | None = None
    created_at: datetime
    expires_at: datetime | None
```

#### 3.1.3 SessionState 模型（`state.py`）

```python
from pydantic import BaseModel

class LoadedSkillInfo(BaseModel):
    """已启用技能的快照信息（带版本）"""
    skill_id: str
    version: str                                       # 启用时的版本号
    current_stage: str | None = None                   # 当前阶段（无 stage 则为 None）
    last_used_at: datetime | None = None               # 最后使用时间（用于 LRU 回收）

class SessionState(BaseModel):
    """会话级治理状态"""
    session_id: str
    skills_metadata: dict[str, SkillMetadata] = {}     # 技能索引（会话级快照）
    skills_loaded: dict[str, LoadedSkillInfo] = {}     # 已启用技能 skill_id → 快照信息（带版本）
    active_tools: list[str] = []                       # 当前可见工具列表
    active_grants: dict[str, Grant] = {}               # skill_id → Grant
    created_at: datetime
    updated_at: datetime
```

#### 3.1.4 Policy 模型（`policy.py`）

```python
from pydantic import BaseModel

class SkillPolicy(BaseModel):
    """单个技能的策略配置"""
    skill_id: str
    auto_grant: bool = True                # 是否自动授权
    require_reason: bool = False           # 是否需要理由
    max_ttl: int = 7200                    # 最大 TTL
    approval_required: bool = False        # 是否需要用户审批

class GovernancePolicy(BaseModel):
    """全局治理策略"""
    default_risk_thresholds: dict[str, str] = {
        "low": "auto",       # 自动授权
        "medium": "reason",  # 需要理由
        "high": "approval"   # 需要审批
    }
    default_ttl: int = 3600
    default_scope: str = "session"
    skill_policies: dict[str, SkillPolicy] = {}   # skill_id → 策略
    blocked_tools: list[str] = []                  # 全局禁用工具
```

### 3.2 核心模块（`src/tool_governance/core/`）

#### 3.2.1 skill_indexer.py

**职责**：扫描 skills/ 目录，解析 YAML frontmatter，构建技能索引。

**核心接口**：

```python
class SkillIndexer:
    def __init__(self, skills_dir: str, cache: TTLCache):
        """初始化，指定 skills 目录和缓存"""

    def build_index(self) -> dict[str, SkillMetadata]:
        """扫描目录，解析所有 SKILL.md，返回 skill_id → SkillMetadata"""

    def list_skills(self) -> list[SkillMetadata]:
        """返回所有技能的元数据摘要"""

    def read_skill(self, skill_id: str) -> SkillContent | None:
        """读取指定技能的完整内容（缓存优先）"""
```

**实现要点**：
- 使用 `yaml.safe_load` 解析 frontmatter，缺字段跳过
- 文件大小限制（默认 100KB），超过则截断或跳过
- 解析失败记录日志但不影响其他技能
- 元数据缓存在 SessionState 中（会话级），文档内容使用 TTLCache（进程级）

#### 3.2.2 state_manager.py

**职责**：管理会话级治理状态的创建、加载、更新、持久化。

**核心接口**：

```python
class StateManager:
    def __init__(self, store: SQLiteStore):
        """初始化，注入持久化存储"""

    def load_or_init(self, session_id: str) -> SessionState:
        """加载已有状态或创建新状态"""

    def save(self, state: SessionState) -> None:
        """持久化状态到 SQLite"""

    def add_to_skills_loaded(self, state: SessionState, skill_id: str) -> None:
        """将技能加入已启用列表"""

    def remove_from_skills_loaded(self, state: SessionState, skill_id: str) -> None:
        """从已启用列表移除技能"""

    def get_active_tools(self, state: SessionState) -> list[str]:
        """返回当前 active_tools"""
```

**实现要点**：
- 状态以 JSON 序列化存入 SQLite
- 每次更新状态时同步更新 `updated_at` 时间戳
- `active_tools` 不做增量维护，每次基于当前 `skills_loaded` 重算

#### 3.2.3 policy_engine.py

**职责**：根据策略配置评估技能是否可以启用。

**核心接口**：

```python
class PolicyEngine:
    def __init__(self, policy: GovernancePolicy):
        """初始化，加载策略配置"""

    def evaluate(
        self,
        skill_id: str,
        skill_meta: SkillMetadata,
        state: SessionState,
        reason: str | None = None
    ) -> PolicyDecision:
        """评估是否允许启用技能，返回决策"""

    def is_tool_allowed(
        self,
        tool_name: str,
        state: SessionState
    ) -> bool:
        """检查某个工具是否在当前 active_tools 中"""
```

```python
class PolicyDecision(BaseModel):
    allowed: bool
    decision: Literal["auto", "reason_required", "approval_required", "denied"]
    reason: str | None = None
```

**评估逻辑**：
1. 检查技能是否在全局黑名单中 → 是则直接 denied
2. 检查是否有 skill-specific 策略 → 优先使用
3. 根据 risk_level 匹配默认阈值：low → auto, medium → reason, high → approval
4. 如果 decision 是 reason_required 但未提供 reason → denied
5. 如果 decision 是 approval_required → 返回需用户确认（由 Hook 层处理）

#### 3.2.4 prompt_composer.py（新增）

**职责**：生成注入模型上下文的 prompt 片段。对应 Skill-Hub 项目中的 PromptComposer，在每轮 UserPromptSubmit 时调用。

**核心接口**：

```python
class PromptComposer:
    def __init__(self, indexer: SkillIndexer):
        """初始化"""

    def compose_context(self, state: SessionState) -> str:
        """生成完整的 additionalContext，包含以下部分：
        1. 技能目录摘要（未启用的技能只显示名称和描述）
        2. 当前已启用技能及其 active_tools
        3. 当前阶段信息（如有 stage）
        4. 授权引导提示（如需更多能力，请先 read_skill → enable_skill）
        """

    def compose_skill_catalog(self, state: SessionState) -> str:
        """只生成技能目录摘要部分（用于 SessionStart 首次注入）"""

    def compose_active_tools_prompt(self, state: SessionState) -> str:
        """生成当前可用工具说明，引导模型正确使用"""
```

**实现要点**：
- 输出控制在 800 字符以内（技能目录摘要 + 活跃工具提示 + 引导语）
- 已启用技能显示更多细节（当前 stage、allowed_tools 列表）
- 未启用技能只显示名称、描述和风险等级
- 每轮 UserPromptSubmit 调用时使用缓存的 skills_metadata，避免重复构建

#### 3.2.5 tool_rewriter.py

**职责**：根据 skills_loaded + current_stage 计算 active_tools。

**核心接口**：

```python
class ToolRewriter:
    def __init__(self, indexer: SkillIndexer):
        """初始化"""

    def recompute_active_tools(self, state: SessionState) -> list[str]:
        """遍历 skills_loaded，按 current_stage 取对应的 allowed_tools 并集，过滤黑名单。
        若技能无 stage 定义，取整个 skill 的 allowed_tools。
        若技能有 stage 定义但 current_stage 为 None，取第一个 stage 的 allowed_tools。"""

    def get_stage_tools(self, skill_meta: SkillMetadata, current_stage: str | None) -> list[str]:
        """获取指定技能在指定阶段的 allowed_tools"""
```

**实现要点**：
- `active_tools` = 元工具 ∪ ⋃{get_stage_tools(skill, stage) | skill ∈ skills_loaded} - blocked_tools
- 元工具始终可见，不受 skills_loaded 影响
- **每轮基于当前状态重算，不做增量 append**（对应 Skill-Hub 设计原则）
- 若技能有 stages 定义，根据 `LoadedSkillInfo.current_stage` 取对应阶段的 allowed_tools
- 每次重算后更新 `state.active_tools`

#### 3.2.5 grant_manager.py

**职责**：管理授权记录的生命周期。

**核心接口**：

```python
class GrantManager:
    def __init__(self, store: SQLiteStore):
        """初始化"""

    def create_grant(
        self,
        session_id: str,
        skill_id: str,
        allowed_ops: list[str],
        scope: str,
        ttl: int,
        granted_by: str,
        reason: str | None
    ) -> Grant:
        """创建新的授权记录"""

    def revoke_grant(self, grant_id: str) -> None:
        """撤销指定授权"""

    def cleanup_expired(self, session_id: str) -> list[str]:
        """清理过期授权，返回被清理的 skill_id 列表"""

    def get_active_grants(self, session_id: str) -> list[Grant]:
        """获取会话中所有活跃授权"""

    def is_grant_valid(self, session_id: str, skill_id: str) -> bool:
        """检查指定技能的授权是否仍然有效"""
```

### 3.3 存储模块（`src/tool_governance/storage/`）

#### 3.3.1 sqlite_store.py

**职责**：基于 SQLite 的持久化存储，数据库文件位于 `${CLAUDE_PLUGIN_DATA}/governance.db`。

**数据表**：

```sql
-- 会话状态表
CREATE TABLE sessions (
    session_id TEXT PRIMARY KEY,
    state_json TEXT NOT NULL,           -- SessionState 序列化 JSON
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

-- 授权记录表
CREATE TABLE grants (
    grant_id TEXT PRIMARY KEY,
    session_id TEXT NOT NULL,
    skill_id TEXT NOT NULL,
    allowed_ops TEXT NOT NULL,           -- JSON array
    scope TEXT NOT NULL,
    ttl_seconds INTEGER NOT NULL,
    status TEXT NOT NULL DEFAULT 'active',
    granted_by TEXT NOT NULL,
    reason TEXT,
    created_at TEXT NOT NULL,
    expires_at TEXT
);

-- 审计日志表 (append-only)
CREATE TABLE audit_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT NOT NULL,
    session_id TEXT NOT NULL,
    event_type TEXT NOT NULL,            -- skill.list / skill.read / skill.enable / tool.call 等
    skill_id TEXT,
    tool_name TEXT,
    decision TEXT,                       -- allow / deny / ask
    detail TEXT,                         -- JSON 额外信息
    created_at TEXT NOT NULL
);

CREATE INDEX idx_grants_session ON grants(session_id, status);
CREATE INDEX idx_audit_session ON audit_log(session_id);
CREATE INDEX idx_audit_event ON audit_log(event_type);
```

### 3.4 缓存模块（`src/tool_governance/utils/cache.py`）

**两层缓存策略**（对应 Skill-Hub 项目实践）：

| 缓存层 | 存储位置 | 内容 | 失效策略 |
|--------|----------|------|----------|
| 元数据缓存 | SessionState.skills_metadata | 技能索引（name、description、allowed_tools 等） | 会话级：新会话重建，显式 refresh |
| 文档缓存 | 进程内 TTLCache | read_skill 解析结果（SOP 正文） | TTL（默认 300s）+ LRU（默认 maxsize=100） |

```python
from cachetools import TTLCache

# 文档缓存：TTL 5分钟，最多缓存 100 个技能文档
skill_doc_cache = TTLCache(maxsize=100, ttl=300)
```

This skill manages progressive disclosure and authorization of capabilities.

## Workflow

1. Use `list_skills` to discover available skills and their risk levels
2. Use `read_skill(skill_id)` to understand a skill's SOP, boundaries, and allowed tools
3. Use `enable_skill(skill_id)` to explicitly authorize a skill for the current session

@tool
def list_skills_tool() -> list[dict]:

    content = indexer.read_skill(skill_id)

@tool
def enable_skill_tool(
    skill_id: str,
    reason: str = "",
    scope: str = "session",
    ttl: int = 3600
) -> dict:
    """Enable a skill for the current session."""
    state = get_current_state()
    decision = policy_engine.evaluate(skill_id, state)
    if not decision.allowed:
        return {"granted": False, "reason": decision.reason}
    grant = grant_manager.create_grant(...)

## 6 关键流程详细设计

### 6.1 SessionStart 处理流程

```python
def handle_session_start(input_data: dict) -> dict:
    session_id = input_data.get("session_id", generate_session_id())

    # 1. 加载或初始化状态
    state = state_manager.load_or_init(session_id)

    # 2. 回收过期 Grant
    expired_skills = grant_manager.cleanup_expired(session_id)
    for skill_id in expired_skills:
        state_manager.remove_from_skills_loaded(state, skill_id)

    # 3. 构建/刷新技能索引
    if not state.skills_metadata:
        state.skills_metadata = skill_indexer.build_index()

    # 4. 重算 active_tools
    state.active_tools = tool_rewriter.recompute_active_tools(state)

    # 5. 持久化状态
    state_manager.save(state)

    # 6. 生成 additionalContext
    summary = tool_rewriter.generate_context_summary(state)

    return {
        "additionalContext": summary
    }
```

### 6.2 UserPromptSubmit 处理流程（对应 wrap_model_call）

> **此 hook 是整个治理机制的核心控制点**，对应 Skill-Hub 中 `wrap_model_call` 的职责。
> 每轮用户消息后触发，负责：1) 重算 active_tools；2) 注入技能状态到 additionalContext。

```python
def handle_user_prompt_submit(input_data: dict) -> dict:
    session_id = input_data.get("session_id")
    state = state_manager.load_or_init(session_id)

    # 1. 回收本轮过期的 grant
    expired_skills = grant_manager.cleanup_expired(session_id)
    for skill_id in expired_skills:
        state_manager.remove_from_skills_loaded(state, skill_id)

    # 2. 基于最新 skills_loaded + current_stage 重算 active_tools（每轮重算，非 append）
    state.active_tools = tool_rewriter.recompute_active_tools(state)

    # 3. 使用 PromptComposer 生成 additionalContext
    context = prompt_composer.compose_context(state)

    # 4. 持久化状态
    state_manager.save(state)

    # 5. 记录审计
    audit_log.record("prompt.submit", session_id,
                     detail={"active_skills": list(state.skills_loaded.keys()),
                             "active_tools_count": len(state.active_tools)})

    return {
        "additionalContext": context
    }
```

**compose_context 输出示例**：

```
[Tool Governance] 当前会话状态：
- 可用技能：db-ops (已启用, stage: analysis), code-edit (未启用), verify (未启用)
- 当前活跃工具：sql_query, logs_search, slow_query_analysis
- 如需更多能力，请先 list_skills → read_skill → enable_skill
```

### 6.3 PreToolUse 处理流程

```python
def handle_pre_tool_use(input_data: dict) -> dict:
    session_id = input_data.get("session_id")
    tool_name = input_data.get("tool_name")

    state = state_manager.load_or_init(session_id)

    # 检查是否是元工具（始终放行）
    META_TOOLS = {"list_skills", "read_skill", "enable_skill",
                  "disable_skill", "grant_status", "run_skill_action",
                  "change_stage", "refresh_skills"}
    # MCP 工具名格式: mcp__tool-governance__<tool_name>
    short_name = extract_tool_name(tool_name)

    if short_name in META_TOOLS:
        return {
            "hookSpecificOutput": {
                "hookEventName": "PreToolUse",
                "permissionDecision": "allow"
            }
        }

    # 检查工具是否在 active_tools 中
    if policy_engine.is_tool_allowed(tool_name, state):
        # 记录审计
        audit_log.record("tool.call", session_id, tool_name=tool_name, decision="allow")
        return {
            "hookSpecificOutput": {
                "hookEventName": "PreToolUse",
                "permissionDecision": "allow"
            }
        }
    else:
        audit_log.record("tool.call", session_id, tool_name=tool_name, decision="deny")
        return {
            "hookSpecificOutput": {
                "hookEventName": "PreToolUse",
                "permissionDecision": "deny",
                "permissionDecisionReason": f"Tool '{tool_name}' is not in active_tools. "
                    "Please use read_skill and enable_skill to authorize the required skill first.",
                "additionalContext": "To use this tool, first discover available skills "
                    "with list_skills, then read_skill to understand the workflow, "
                    "then enable_skill to authorize."
            }
        }
```

> **[不确定项]** `session_id` 在 hook 输入中的具体字段名可能因 Claude Code 版本而异。需在实现时根据实际 hook 输入 schema 确认。如果 hook 输入中没有 session_id，可能需要使用其他标识符（如 conversation_id）或自行生成并通过文件/SQLite 关联。

## 8 安全设计

### 8.1 输入安全

- SKILL.md 作为不可信输入：
  - `yaml.safe_load()` 解析 frontmatter
  - 文件大小限制（默认 100KB）
  - 描述字段截断（默认 500 字符）
  - 解析失败只告警不影响其他技能

### 8.2 权限安全

- 元工具（list_skills 等）始终可用，不做权限拦截
- 底层工具调用必须通过 PreToolUse hook 检查 active_tools
- 高风险技能（risk_level=high）的 enable_skill 需要用户确认
- 策略配置文件不可通过 MCP 工具修改，只能通过文件编辑

### 8.3 数据安全

- SQLite 数据库存储在 `${CLAUDE_PLUGIN_DATA}` 目录，遵循 Claude Code 的数据隔离
- 审计日志表 append-only（不提供 DELETE/UPDATE 接口）
- Grant 中的 reason 字段不存储敏感信息

### 10-A.4 实施偏差记录

| 偏差项 | 原设计 | 实际实现 | 原因 |
|-------|--------|---------|------|
| requires-python | >=3.11 | >=3.10 | 开发机 Python 版本为 3.10.13，降低门槛兼容当前环境 |
| dev 依赖版本 | 开放范围 (ruff>=0.1) | 收窄范围 (ruff>=0.8,<1.0) | pip 解析开放范围时严重回溯，收窄后秒装 |
| hooks.json 格式 | 顶层直接是事件 map（S4.3） | 需 `{description, hooks: {事件 map}}` 包装 | `claude plugin validate` 要求 oJ8 schema：plugin hooks 有外层 object |
